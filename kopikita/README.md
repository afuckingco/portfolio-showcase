## What is this?

A complete business analytics project built around a fictional Bali cafe.
The data is **synthetic** (deterministic seed) but follows realistic patterns:

- 24-item menu across 6 categories (Coffee, Tea, Juice, Food, Pastry, AddOn)
- Prices in IDR (Bali cafe-realistic: Rp 10k-38k)
- Daily/weekly seasonality (morning rush, lunch peak, afternoon coffee, dinner)
- Weekend boost, holiday dips, weather proxy (rainy day penalty)
- 120 regulars + 4,000 walk-in pseudo customers
- 6 months (Jan-Jun 2026) → 11,500 transactions, 20,650 line items

> 💡 Built as a portfolio project — every script is runnable on your machine
> with the data the generator creates. No external APIs, no scraping.

---

## Pipeline

```
1. generate_data.py    → synthetic POS data (deterministic seed=42)
2. eda.py              → 6 exploratory charts + daily summary
3. forecast.py         → Prophet daily-revenue forecast + MAPE eval
4. segment.py          → RFM + KMeans customer segmentation
5. anomaly.py          → Z-score + Isolation Forest anomaly detection
```

All scripts write to `outputs/` (CSV summaries + PNG charts).

## Quick Start

```bash
# 1. Install dependencies
pip install -r requirements.txt

# 2. Generate synthetic data (creates data/transactions.csv)
python src/generate_data.py

# 3. Run the full analytics pipeline
python src/eda.py        # → outputs/charts/01-06*.png
python src/forecast.py   # → outputs/charts/07-08*.png, predictions/forecast.csv
python src/segment.py    # → outputs/charts/09-10*.png, customer_segments.csv
python src/anomaly.py    # → outputs/charts/11*.png, anomalies.csv
```

Outputs land in `outputs/charts/` (PNG) and `outputs/` (CSV summaries).

## Results (synthetic data)

### EDA highlights
- **181 days**, **Rp 739M** total revenue, avg **Rp 4.08M/day**
- **Top items:** Nasi Goreng, Ayam Geprek, Es Kopi Gula Aren, Es Kopi Susu, Latte
- **Median basket:** Rp 56,000 (p90: Rp 121,000)

### Forecast (`forecast.py`)
- Prophet model with daily + weekly seasonality (multiplicative mode)
- Trained on Jan–May (151 days), tested on Jun (30 days)
- **MAPE: 16.86%** · MAE: Rp 613,102
- 30-day forward projection: avg Rp 4.81M/day
- Caveat: synthetic data has injected anomalies (~3% of days) which inflate MAPE

### Customer segments (`segment.py`)
- RFM on 120 regular customers → KMeans (k=4)
- Auto-labeled segments: **Champions**, **At Risk**, **New / Occasional**, **Dormant**
- Champions contribute the largest revenue share despite being fewer in count

### Anomaly detection (`anomaly.py`)
- Z-score (rolling 14-day, |z|>2): 5 flagged days (2.8%)
- Isolation Forest (multivariate, 5% contamination): 9 flagged days (5.0%)
- Overlap: 1 day (high-confidence anomaly)
- Union: 13 days worth investigating

## Methods — Honest Notes

**Forecasting:**
- Prophet is chosen over LSTM because it's interpretable (decomposable trend +
  seasonality), handles missing days well, and trains in seconds. LSTM would
  likely overfit on 5 months of training data.
- The 16.86% MAPE reflects that synthetic data has injected anomalies (low-traffic
  days, holiday dips). On clean real data, expect < 10%.

**Segmentation:**
- Forced k=4 instead of silhouette-optimal k=2 because customer segmentation
  is more useful with 4 named segments for marketing actions.
- Synthetic data has limited recency spread (most regulars are recent), so
  the "At Risk" segment is small. With real cafe data spanning 2+ years,
  segment differentiation would be stronger.

**Anomaly detection:**
- Both methods have value: z-score catches univariate revenue shocks,
  Isolation Forest catches multivariate days that look unusual across
  revenue+transactions+items even when each metric alone is normal.
- We mark overlap (both methods agree) as high-confidence anomalies.

## Why this exists

This was built as a portfolio piece to demonstrate practical end-to-end ML on
a realistic small-business dataset. The synthetic data was designed to have
enough quirks (rainy days, holiday dips, walk-in vs regular customers) that the
analysis surface is non-trivial — but small enough to run on a laptop in seconds.

**If applied to a real cafe**, this same pipeline would:
- Help the owner forecast daily prep needs (reduce food waste)
- Identify loyal customers for a loyalty-program rollout
- Flag unexpected slow days for investigation (staffing, marketing)
- Drive menu engineering decisions (which items to keep / retire)
